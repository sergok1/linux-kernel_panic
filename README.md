
### Что произошло

На экране видно сообщение:
```text
KERNEL PANIC
Please reboot your computer.
VFS: Unable to mount root fs on unknown-block(0,0)
```
Это означает, что ядро Linux не может найти или смонтировать корневую файловую систему. 
Обычно это происходит после обновления системы или ядра, если что-то пошло не так с загрузчиком, драйверами или разметкой диска.



Долгий анализ, привел к выводу:
Новый kernel 6.14.0-27 собрал «кривой» initramfs, поэтому при шифрованном корне ядро не находит том LUKS и падает. 
Раз ядро 6.14.0-24 грузится, значит сама разметка/шифрование в порядке надо починить только файловый образ initramfs 
для проблемной версии и, при включённом Secure Boot, убедиться, что подписаны нужные модули.



Скрипт для обновления initramfs и GRUB с указанным ядром:


#### Как использовать:

1. **Сохрани скрипт**

2. **Сделайте его исполняемым**:
   ```bash
   chmod +x fix-kernel.sh
   ```

3. **Запустите с правами root**:
   ```bash
   sudo ./fix-kernel.sh
   ```

#### Что делает скрипт:

- **Показывает список** всех установленных ядер в системе
- **Подсвечивает** последнее (потенциально проблемное) ядро
- **Запрашивает** версию ядра для пересборки initramfs
- **Проверяет** существование указанного ядра
- **Выполняет** `update-initramfs -c -k <версия>`
- **Обновляет** GRUB
- **Спрашивает** о перезагрузке (на случай, если хотите что-то ещё сделать)

Скрипт также включает проверку ошибок на каждом этапе и цветной вывод для удобства!
